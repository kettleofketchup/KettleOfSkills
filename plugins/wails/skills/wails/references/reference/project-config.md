# Project Config (wails.json)

Root-level project configuration file. Generated by `wails init`, consumed by CLI.

## Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Project name |
| `outputfilename` | `string` | Output binary name |
| `frontend:dir` | `string` | Frontend directory relative to project root (default: `frontend`) |
| `frontend:install` | `string` | Command to install frontend dependencies (e.g. `npm install`) |
| `frontend:build` | `string` | Command to build frontend (e.g. `npm run build`) |
| `frontend:dev:watcher` | `string` | Command to run frontend dev server (e.g. `npm run dev`) |
| `frontend:dev:serverUrl` | `string` | URL of frontend dev server (e.g. `http://localhost:5173`). Wails proxies to this instead of serving from assetdir |
| `wailsjsdir` | `string` | Directory for generated wailsjs modules (default: frontend dir) |
| `version` | `string` | Wails project config version |
| `author` | `object` | Author info |
| `author.name` | `string` | Author name |
| `author.email` | `string` | Author email |
| `info` | `object` | Application metadata (used in platform packaging) |
| `info.companyName` | `string` | Company name |
| `info.productName` | `string` | Product name |
| `info.productVersion` | `string` | Product version |
| `info.copyright` | `string` | Copyright string |
| `info.comments` | `string` | Application comments/description |
| `nsisType` | `string` | NSIS installer type: `"multiple"` (per-user, default) or `"single"` (single, admin required) |
| `obfuscated` | `bool` | Default obfuscation via garble |
| `garbleargs` | `string` | Default garble args (e.g. `-literals -tiny -seed=random`) |
| `bindings` | `object` | Bindings generation config |
| `bindings.ts_generation` | `object` | TypeScript generation options |
| `bindings.ts_generation.prefix` | `string` | Prefix for generated TS filenames |
| `bindings.ts_generation.suffix` | `string` | Suffix for generated TS filenames |
| `debounce` | `int` | Default debounce for dev mode hot-reload (ms) |

## Example

```json
{
  "name": "myapp",
  "outputfilename": "myapp",
  "frontend:dir": "frontend",
  "frontend:install": "npm install",
  "frontend:build": "npm run build",
  "frontend:dev:watcher": "npm run dev",
  "frontend:dev:serverUrl": "http://localhost:5173",
  "wailsjsdir": "frontend/src",
  "version": "2",
  "author": {
    "name": "Dev Name",
    "email": "dev@example.com"
  },
  "info": {
    "companyName": "My Company",
    "productName": "My App",
    "productVersion": "1.0.0",
    "copyright": "Copyright 2024 My Company",
    "comments": "A desktop application"
  },
  "nsisType": "multiple",
  "obfuscated": false,
  "garbleargs": "-literals -tiny -seed=random",
  "bindings": {
    "ts_generation": {
      "prefix": "",
      "suffix": ""
    }
  },
  "debounce": 100
}
```

## Dev server flow

When `frontend:dev:watcher` is set, `wails dev`:
1. Runs `frontend:install` (if set)
2. Starts `frontend:dev:watcher` as background process
3. Waits for `frontend:dev:serverUrl` to respond
4. Proxies asset requests to that URL instead of serving from disk
5. Rebuilds Go backend on `.go` file changes

When no `frontend:dev:watcher`, assets served from `assetdir` flag or compiled `embed.FS`.
